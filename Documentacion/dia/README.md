# Guía de Instalación del Proyecto - 10 Días

## 📋 ¿Qué es esta guía?

Esta guía está hecha para personas que **nunca han programado** o instalado proyectos antes. Te llevaré paso a paso, desde instalar los programas necesarios hasta tener el proyecto funcionando en tu computadora.

**No necesitas saber nada de programación para empezar.** Todo está explicado de forma simple.

---

## 🗓️ Plan de 10 Días (muy fácil)

### **[Día 1: Instalar los programas básicos](dia01.md)**
- ✅ Instalar Git (programa para guardar código)
- ✅ Instalar Visual Studio Code (editor de texto especial)
- ✅ Aprender a usar la terminal (ventana negra para escribir comandos)
- ✅ Practicar comandos básicos

**Tiempo que te tomará:** 1-2 horas

---

### **[Día 2: Instalar Node.js](dia02.md)**
- ✅ Instalar Node.js (para ejecutar código JavaScript)
- ✅ Entender qué es npm (descargador de herramientas)
- ✅ Crear tu primer proyecto pequeño
- ✅ Aprender a instalar herramientas extras

**Tiempo que te tomará:** 1-2 horas

---

### **[Día 3: Instalar Docker](dia03.md)**
- ✅ Instalar WSL 2 (sistema Linux para Windows)
- ✅ Instalar Docker Desktop (crea "mini computadoras" virtuales)
- ✅ Entender qué son los contenedores (muy fácil)
- ✅ Comandos básicos de Docker

**Tiempo que te tomará:** 2-3 horas

---

### **[Día 4: Descargar el proyecto](dia04.md)**
- ✅ Descargar el proyecto a tu computadora
- ✅ Ver qué carpetas y archivos tiene
- ✅ Entender para qué sirve cada carpeta
- ✅ Abrir el proyecto en VS Code

**Tiempo que te tomará:** 1-2 horas

---

### **[Día 5: Entender cómo funciona el proyecto](dia05.md)**
- ✅ Ver qué es Docker Compose (organizador de contenedores)
- ✅ Leer el archivo compose.yml (sin miedo)
- ✅ Entender cómo se comunican las partes del proyecto
- ✅ Conceptos simples de redes y almacenamiento

**Tiempo que te tomará:** 2-3 horas

---

### **[Día 6: ¡Encender el proyecto por primera vez!](dia06.md)**
- ✅ Ejecutar el proyecto (un solo comando)
- ✅ Ver que todo funcione correctamente
- ✅ Abrir la página web en tu navegador
- ✅ Arreglar problemas si algo falla

**Tiempo que te tomará:** 1-2 horas

---

### **[Día 7: Ver cómo funciona el servidor](dia07.md)**
- ✅ Explorar el código del servidor (backend)
- ✅ Ver las direcciones web que responde (API)
- ✅ Entender cómo funciona el login
- ✅ Probar el servidor con una herramienta simple

**Tiempo que te tomará:** 2-3 horas

---

### **[Día 8: Ver cómo funciona la página web](dia08.md)**
- ✅ Explorar el código de la página web (frontend)
- ✅ Ver el código de login, registro y perfil
- ✅ Entender cómo la página habla con el servidor
- ✅ Aprender dónde se guardan los datos en el navegador

**Tiempo que te tomará:** 2-3 horas

---

### **[Día 9: Aprender a usar la base de datos](dia09.md)**
- ✅ Conectarse a MySQL (donde se guardan los usuarios)
- ✅ Ver cómo están organizados los datos
- ✅ Aprender comandos simples para ver y cambiar datos
- ✅ Hacer una copia de seguridad de los datos

**Tiempo que te tomará:** 2-3 horas

---

### **[Día 10: Hacer cambios al proyecto](dia10.md)**
- ✅ Agregar algo nuevo al proyecto
- ✅ Aprender a encontrar errores
- ✅ Hacer que el proyecto funcione mejor
- ✅ Preparar el proyecto para mostrarlo a otros

**Tiempo que te tomará:** 3-4 horas

## 🚀 ¿Puedo empezar rápido?

Si ya sabes algo de programación y quieres probar rápido:

```bash
# 1. Descargar el proyecto
git clone URL_DEL_REPOSITORIO
cd Practicas-Castelar

# 2. Asegúrate de que Docker Desktop esté abierto

# 3. Iniciar todo
docker compose up -d --build

# 4. Abre tu navegador:
# Página web: http://localhost:8080
# Servidor: http://localhost:3001
```

**👉 Recomendación:** Si nunca has programado, **NO uses el inicio rápido**. Sigue la guía día por día para entender todo bien.

## 🆘 ¿Necesitas ayuda?

### Problemas comunes y soluciones

**1. Docker no arranca**
- Solución: Abre Docker Desktop y espera a que termine de cargar
- Ve al [Día 3](dia03.md) para más ayuda

**2. Dice "puerto en uso"**
- Solución: Otro programa está usando ese puerto
- Ve al [Día 5](dia05.md) para cambiar el puerto

**3. El servidor no se conecta a la base de datos**
- Solución: Espera un poco más, MySQL tarda en iniciarse
- Ve al [Día 6](dia06.md) para revisar logs

**4. Los cambios no aparecen**
- Solución: Reinicia todo con `docker compose up -d --build`
- Ve al [Día 10](dia10.md) para más info
